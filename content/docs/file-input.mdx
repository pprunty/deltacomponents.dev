# FileInput

A versatile file input component with drag-and-drop support, file previews, validation, and multiple style variants.

## Basic Usage

## Installation

```bash
npx shadcn@latest add "file-input"
```

## Examples

### Default Variant

```tsx
<FileInput
  label="Upload Document"
  name="document"
  description="Upload a file for your application"
  accept=".pdf,.docx,.jpg,.png"
  hint="Accepted formats: PDF, DOCX, JPG, PNG"
/>
```

### Multiple Files Upload

```tsx
<FileInput
  label="Upload Images"
  name="images"
  multiple={true}
  maxFiles={3}
  accept="image/*"
  hint="Upload up to 3 images"
/>
```

### Pill Variant with Colored Border

```tsx
<FileInput
  label="Upload Document"
  name="document"
  variant="pill"
  coloredBorder={true}
  accept=".pdf,.docx"
  hint="Upload your resume or cover letter"
/>
```

## Validation

The FileInput component supports Zod validation:

```tsx
import { z } from "zod";
import { useState } from "react";
import { FileInput } from "@/components/ui/file-input";

export default function FileValidationExample() {
  const [error, setError] = useState<string | undefined>();
  
  // Schema to validate file size
  const maxSize = 5 * 1024 * 1024; // 5MB
  const schema = z.instanceof(File).refine(
    (file) => file.size <= maxSize,
    {
      message: "File must be less than 5MB",
    }
  );
  
  const handleValidate = (isValid: boolean, file: File, error?: string) => {
    setError(error);
  };
  
  return (
    <FileInput
      label="Upload Image"
      name="image"
      accept="image/*"
      schema={schema}
      onValidate={handleValidate}
      error={error}
      hint="Maximum file size: 5MB"
    />
  );
}
```

## Accessibility

The FileInput component is built with accessibility in mind:

- Proper labeling and ARIA attributes
- Keyboard navigable interface
- Error messages are properly associated with the input
- Support for screen readers with appropriate text descriptions
- Visual indicators for drag-and-drop states
