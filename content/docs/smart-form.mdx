## Field Types

The Smart Form component supports various field types:

- Text fields (text, email, password, number, tel, url)
- Checkbox
- Select
- Radio
- Switch
- Textarea
- Date
- File
- OTP (One-Time Password)
- Tags Input
- Custom fields

## Tags Input Example

Here's an example of using the TagsInput component in a Smart Form:

```tsx
import { SmartForm } from "@/components/ui/smart-form"
import { z } from "zod"

const schema = z.object({
  tags: z.array(z.string()).min(1, "At least one tag is required").max(5, "Maximum 5 tags allowed")
})

const fields = [
  {
    type: "tags",
    name: "tags",
    label: "Tags",
    description: "Add tags by pressing space, comma, or enter",
    hint: "Press backspace to remove the last tag",
    variant: "pill",
    triggerKey: "Enter", // Optional: can be "Enter", "Space", or "Comma"
    required: true
  }
]

export default function TagsFormExample() {
  return (
    <SmartForm
      fields={fields}
      schema={schema}
      onSubmit={(data) => console.log(data)}
    />
  )
}
```

## Validation

Form validation is handled using Zod schemas. The component automatically validates fields on change and submission.

## Layout Options

The form supports three layout options:
- Vertical (default)
- Horizontal
- Grid

## Custom Styling

You can customize the form's appearance using Tailwind CSS classes:

```tsx
<SmartForm
  className="max-w-md mx-auto"
  fieldClassName="mb-4"
  submitClassName="w-full"
/>
```

## Accessibility

The Smart Form component follows WAI-ARIA guidelines and includes proper ARIA attributes for accessibility. All form fields are properly labeled and error messages are associated with their respective fields.
