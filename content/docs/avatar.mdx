# Avatar [#avatar]

A versatile avatar component that supports images and initials fallback. Features automatic gradient generation based on names and image caching for better performance.

## Features

- **Image Support**: Display profile pictures with automatic fallback
- **Initials Display**: Show user initials when no image is available
- **Gradient Generation**: Automatic gradient generation based on user names
- **Image Caching**: Built-in image caching for better performance
- **Size Customization**: Flexible sizing options
- **Interactive**: Optional click handler with hover effects
- **Accessibility**: Proper ARIA attributes and keyboard navigation

## Usage Examples

### Basic Avatar

```tsx
import { Avatar } from "@/components/ui/avatar"

export default function AvatarDemo() {
  return (
    <Avatar 
      src="https://github.com/username.png"
      name="John Doe"
      size={40}
    />
  )
}
```

### Size Variants

```tsx
import { Avatar } from "@/components/ui/avatar"

export default function AvatarDemo() {
  return (
    <div className="flex items-center gap-4">
      <Avatar name="Small" size={24} />
      <Avatar name="Medium" size={40} />
      <Avatar name="Large" size={64} />
    </div>
  )
}
```

### Fallback Variants

```tsx
import { Avatar } from "@/components/ui/avatar"

export default function AvatarDemo() {
  return (
    <div className="flex items-center gap-4">
      <Avatar name="John Doe" size={40} />
      <Avatar name="Jane" size={40} />
      <Avatar size={40} />
    </div>
  )
}
```

### Interactive Variants

```tsx
import { Avatar } from "@/components/ui/avatar"

export default function AvatarDemo() {
  return (
    <div className="flex items-center gap-4">
      <Avatar 
        name="Click Me"
        size={40}
        onClick={() => console.log('Avatar clicked!')}
      />
      <Avatar 
        name="No Image"
        size={40}
        onClick={() => console.log('Fallback avatar clicked!')}
      />
    </div>
  )
}
```

## Accessibility

The Avatar component follows accessibility best practices:

- Includes proper `alt` text for images
- Uses semantic HTML elements
- Provides fallback content for screen readers
- Maintains proper contrast ratios
- Supports keyboard navigation
- Adds proper ARIA roles and keyboard support when interactive
