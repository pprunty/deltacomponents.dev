## Features

- Customizable size and shape variants
- Built-in validation with Zod integration
- Accessible with proper ARIA attributes
- Extended click area for better mobile usability
- Custom colors for active state
- Support for descriptions and hints
- Error handling and display
- Loading state support
- Required field validation

## Usage

```tsx
import { SwitchInput } from "@/delta/components/switch-input"
import { z } from "zod"

export default function SwitchInputDemo() {
  return (
    <div className="space-y-4">
      <SwitchInput 
        label="Enable Notifications"
        name="notifications"
        description="Receive updates about your account"
        defaultChecked
      />
      <SwitchInput 
        label="Dark Mode"
        name="dark-mode"
        variant="pill"
        hint="Toggle between light and dark theme"
      />
      <SwitchInput 
        label="Accept Terms"
        name="terms"
        required
        schema={z.literal(true, {
          errorMap: () => ({ message: 'You must accept the terms' }),
        })}
      />
    </div>
  )
}
```

## Custom Styling

You can customize the switch appearance using Tailwind CSS classes:

```tsx
<SwitchInput 
  label="Custom Style"
  name="custom"
  className="bg-blue-500"
  activeColor="bg-purple-500"
/>
```

## Accessibility

The SwitchInput component follows WAI-ARIA guidelines and includes:
- Proper ARIA attributes for switch role
- Keyboard navigation support
- Screen reader compatibility
- Error message association
- Required field indication
- Loading state indication
