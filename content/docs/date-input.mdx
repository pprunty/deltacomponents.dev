# DateInput

A flexible date input component with an integrated calendar picker, validation support, and customizable styling options.

## Basic Usage

## Installation

```bash
npx shadcn@latest add "date-input"
```

## Examples

### Default Variant

```tsx
<DateInput
  label="Event Date"
  name="event-date"
  description="Select the date of your event"
  placeholder="Select a date"
/>
```

### Pill Variant with Colored Border

```tsx
<DateInput
  label="Event Date"
  name="event-date"
  variant="pill"
  coloredBorder={true}
  placeholder="Select a date"
/>
```

### With Min and Max Date Constraints

```tsx
<DateInput
  label="Appointment Date"
  name="appointment-date"
  minDate={new Date()} // Today's date
  maxDate={new Date(Date.now() + 90 * 24 * 60 * 60 * 1000)} // 90 days from now
  hint="Select a date within the next 90 days"
/>
```

## Validation

The DateInput component supports Zod validation:

```tsx
import { z } from "zod";
import { useState } from "react";
import { DateInput } from "@/components/ui/date-input";

export default function DateValidationExample() {
  const [error, setError] = useState<string | undefined>();
  
  // Schema to validate the date is in the future
  const schema = z.date().min(new Date(), {
    message: "Date must be in the future",
  });
  
  const handleValidate = (isValid: boolean, value: Date, error?: string) => {
    setError(error);
  };
  
  return (
    <DateInput
      label="Event Date"
      name="event-date"
      schema={schema}
      onValidate={handleValidate}
      error={error}
      hint="Date must be in the future"
    />
  );
}
```

## Accessibility

The DateInput component is built with accessibility in mind:

- Proper labeling and ARIA attributes
- Keyboard navigation support in the calendar
- Error messages are properly associated with inputs
- Clear focus states for keyboard users
